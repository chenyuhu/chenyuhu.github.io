2019-05-30
PC端, 前端, 学习

vue-cli3

🛠️ Vue.js 开发的标准工具 - 尤雨溪

工具链

  • @vue/cli: 搭建交互式的项目脚手架
  • @vue/cli-service-global: 通过 @vue/cli + @vue/cli-service-global 快速开始零配置原型开发

初始化项目

  • Please pick a preset: 你可以选默认的包含了基本的 Babel + ESLint 设置的 preset，也可以选“手动选择特性”来选取需要的特性。默认的设置非常适合快速创建一个新项目的原型，而手动设置则提供了更多的选项，它们是面向生产的项目更加需要的

  • Check the features needed for your project?：选择你的项目需要的特性，值的解释的是，PWA 是谷歌的Web App，Unit是单元测试、e2e是端到端测试，他们的区别请参考unit单元测试和e2e端到端测试的区别

  • TypeScript

    • Use class-style component syntax?: 是否使用class风格的组件语法
    • Use Babel alongside TypeScript for auto-detected polyfills?：是否使用babel做转义

  • Router

    • Use history mode for router? (Requires proper server setup for index fallback in production): 使用 history 路由模式需要服务器配合，参考 SPA F5刷新页面后404 和 HTML5 History 模式

  • CSS pre-processor

    • Pick a CSS pre-processor (PostCSS, Autoprefixer and CSS Modules are supported by default): 选择一个 CSS 预处理器
      • Sass/SCSS
      • Less
      • Stylus

  • Linter / Formatter

    • Pick a linter / formatter config: 选择一个 linter和formatter的配置，建议选择基础的，然后根据自己的习惯去配置
      • TSLint
      • ESLint with error prevention only
      • ESLint + Airbnb config
      • ESLint + Standard config
      • ESLint + Pretiier
    • Pick additional lint features?: 选择额外的 lint 特性
      • Lint on save: 保存就检测
      • Lint and fix on commit: fix和commit时候检查,天下程序一大抄，尤雨溪最会抄。这块的详细描述，请参考记一次gitHook带来的思考🤔

  • Unit Testing: 单元测试

    • Pick a unit testing solution: 选择一个单元测试的解决方案
      • Mocha + Chai
      • Jest

  • E2E Testing：端到端测试

    • Pick a E2E testing solution: 选择一个端到端测试方案
      • Cypress（Chrome only）
      • Nightwatch（S饿了nium-based）

  • Where do you prefer placing config for Babel, PostCSS, ESLint, etc.? : 配置文件如何防止

    • In dedicated config files: 使用专用的配置文件
    • In package.json: 全部放到 package.json

  • Save this as a preset for future projects?: 是否把本次选择的配置当作模版

electron

安装依赖

  • electron-builder: A Vue Cli 3 plugin for Electron with no required configuration that uses Electron Builder.
  • electron: Electron构建跨平台应用Mac/Windows/Linux
  • electron-debug: electron的代码调试
  • vue-electron: 一个把 electron API 带进 Vue对象的 vue 插件
  • vuex-electron: 最简单的在不同进程（包括主进程）间共享你的 Vuex Store的方式
  • electron-updater: electron 热更新插件
  • electron-log：一个向本地写日志文件的插件
  • electron-is-dev: 检查 Electron 是不是在 开发环境下运行，然后利用它搞事情
  • electron-devtools-installer: 一个将devtool注入到electron中的简单方式
